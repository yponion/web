 고등학생 시절 R&E 프로젝트를 통해 아두이노와 코딩을 처음 접하게 되었습니다. 그 경험을 통해 컴퓨터 공학에 흥미를 느껴서 대학에서 이 분야를 전공하게 되었습니다. 대학에서 안드로이드 앱 개발을 경험하며 다시 한 번 흥미로운 분야를 발견했습니다. 이로 인해 앱 개발자로 성장하고자 결심하게 되었습니다. 그러던 와중 웹 개발을 경험하니 또 흥미를 느껴 경험을 쌓고 있습니다. 이렇게 관심이 없거나 접해보지 못했던 분야일지라도, 경험 해보면 자신의 적성과 흥미에 맞는 일을 찾을 수 있다는 것을 알게 되었습니다. 그래서 저는 다양한 경험을 쌓기 위해 노력해야 한다는 것을 깨달았습니다. 이에 관심은 있었지만 시도해보지 않았던 iOS 앱 개발 공부를 시작해보려고 합니다. 이렇게 다양한 경험을 통해 더욱 풍부한 개발자로 성장하고자 합니다.

스티브 잡스, CEO가 말하는 인문학

개발자란 소프트웨어 및 애플리케이션을 개발하는 전문가로서, 기술적인 능력과 문제 해결 능력을 갖추고, 협업과 커뮤니케이션 능력을 가지며, 지속적인 학습과 업데이트를 통해 기술을 발전시키는 사람입니다.

 과거에는 개발자는 단순히 개발 역량만 갖추면 된다고 생각했지만, 스티브 잡스의 연설과 책을 접하면서 이 생각이 180도 달라졌습니다. 개발자에게는 인문학적인 이해도가 필수라고 깨달았습니다. 예를 들어, 물건을 구매할 때 기능이 완벽히 동일하다면 대부분의 사람들은 저렴한 제품을 선택할 것입니다. 하지만 디자인 요소가 추가된다면 사람들은 당연히 더 아름다운 제품을 선호합니다. 이로부터 왜 스티브 잡스가 대학에서 서체 수업을 도강까지 하며 들었는지 이해하게 되었습니다. 개발 과정에서 사람들을 이해하지 못한 결과물은 시장에서 무시받지만, 사용자의 니즈를 반영한 제품은 열렬한 호응을 얻습니다. 따라서 기술 능력은 중요하지만, 최종적으로 사용하는 것은 사람이기 때문에 인문학적인 이해도의 중요성을 강조합니다. 그래서 스티브 잡스의 말처럼 과학 기술과 인문학의 교차점에 위치하며, 인간 중심적인 접근 방식을 가질 것입니다.

자율주행, 사고율과 안정성보다 선행 되어야할 법과제도

규제와 인프라 구축

자율주행을 위해서는 운전 규칙과 교통법규를 적용할 수 있는 법적 기반과 인프라 구축이 필요합니다. 자율주행 차량의 도로 운행과 관련된 규제, 자동차의 기술 안전 기준, 데이터 보호 및 개인정보 보안 등에 대한 법적 프레임워크를 마련해야 합니다. 또한, 도로 인프라에는 자율주행을 지원하는 표지판, 센서 기반의 통신 인프라, 충전 인프라 등이 구축되어야 합니다.

인공지능 윤리와 책임

자율주행 차량이 사람의 의사 결정을 대신하는 경우, 인공지능의 윤리와 책임 문제가 중요해집니다. 자율주행 시스템의 개발자와 제조업체는 윤리적 원칙을 정립하고, 인공지능의 결정 방식을 투명하게 공개해야 합니다. 또한, 사고나 사망 같은 사례에 대한 책임 문제도 명확하게 규정되어야 합니다.

데이터 보호와 개인정보

자율주행 차량은 많은 양의 데이터를 수집하고 처리합니다. 이에 따라 개인정보 보호와 데이터 보호 문제도 해결되어야 합니다. 데이터 수집, 저장 및 공유에 대한 규제와 프라이버시 보호를 위한 법적 조치가 필요합니다.

보험과 책임

자율주행 시스템이 운전을 대신하게 되면, 사고 발생 시 법적인 책임에 대한 문제가 제기됩니다. 자율주행에 따른 보험 제도와 사고 발생 시 책임 분담 방식에 대한 법적 장치가 필요합니다. 이를 통해 피해자 보호와 정확한 책임 추적이 가능해집니다.

국제 표준화와 협력

자율주행은 국경을 넘어서는 기술이므로 국제적인 표준화와 협력이 필요합니다. 자율주행 기술, 데이터 보호, 책임 분담 등에 대한 국제 표준화 노력이 이루어져야 하며, 국가간 협력을 통해 법과제도를 조율해야 합니다.

 위와 같은 법과제도의 선행은 자율주행 기술의 안전성과 사회적 수용성을 보장하기 위해 중요합니다. 자율주행은 인간의 개입 없이 차량이 주행하는 것을 의미하며, 이는 인간 운전자의 오류나 부주의로 인한 사고 가능성을 줄일 수 있습니다. 그러나 자율주행 시스템의 안전성을 보장하기 위해서는 철저한 테스트와 검증, 안전 기준을 갖추는 것이 필요합니다. 법과제도를 통해 자율주행 시스템의 안전성을 검증하고 규제할 수 있으며, 사고 발생 시의 책임과 보상 체계를 마련하여 사용자들의 안전을 확보할 수 있습니다.

 또한, 자율주행 차량은 인공지능이 결정을 대신하게 됩니다. 이로 인해 윤리적인 선택과 책임 문제가 발생할 수 있습니다. 이러한 문제를 해결하기 위해 법과제도는 인공지능의 윤리적 원칙을 규정하고, 책임과 권한을 명확히 분담하여 사회적으로 수용 가능한 자율주행 시스템을 구축할 수 있습니다. 이를 통해 억울한 일을 당하는 사례를 최소화하고 공정한 대우를 보장할 수 있어야 합니다.

 뿐만 아니라, 자율주행 시스템은 많은 양의 데이터를 수집하고 처리합니다. 법과제도는 데이터 수집과 보호, 개인정보 처리에 대한 규제와 프라이버시 보호를 제공함으로써 사용자들의 개인정보와 데이터 보안을 보장할 수 있어야 합니다.

 마지막으로, 법과제도의 선행은 자율주행 기술의 사회적 수용성을 보장하기 위해 중요합니다. 자율주행 기술은 사회에 큰 변화를 가져올 것으로 예상되며, 이러한 변화를 사회적으로 수용하기 위해서는 법과제도가 선행되어야 합니다. 법과제도는 사용자들의 우려와 궁금증에 대한 답변을 제공하고, 자율주행의 장점과 이점을 설명하는 법적 지원을 통해 자율주행 기술을 사회적으로 수용 가능하게 만들 수 있습니다.

 따라서, 법과제도의 선행은 자율주행 기술의 안전성과 사회적 수용성을 보장하기 위해 필수적입니다. 이를 통해 안전한 운행, 윤리적인 선택과 책임, 개인정보와 데이터 보호, 사회적 수용성을 확보할 수 있습니다.

기술적으로 어려웠던 부분과 해결 방안

마우스가 요소 위로 올라갈 때 zIndex 값이 10으로 변경되며, 마우스가 요소를 벗어날 때 원래 값으로 설정하려 하였습니다.

var competence = document.getElementsByClassName("competence");
for(i = 0; i<6; i++){
    competence[i].style.zIndex = i;
}
for(i = 0; i<6; i++){
    competence[i].onmouseover = function(){
        this.style.zIndex = 10;
    }
}
for(i = 0; i<; i++){
    competence[i].onmouseout = function(){
        this.style.zIndex = i;
    }
}

위와 같이 코드를 작성하니 onmouseout 이벤트 핸들러 내부에서 this.style.zIndex = i;를 사용하는 것은 클로저(closure) 문제로 인해 원하는 결과를 얻을 수 없는 상황이였습니다. 클로저 문제란 반복문 내에서 생성된 함수에서 외부 변수에 접근할 때 발생하는 문제로 i 변수는 반복문이 끝난 후에 최종 값인 6이 되어버리기 때문에 onmouseout 이벤트 핸들러에서 항상 6이 적용되게 되는 문제가 발생합니다.

var competence = document.getElementsByClassName("competence");
for (i = 0; i < 6; i++) {
    competence[i].style.zIndex = i;

    competence[i].onmouseover = function() {
    this.style.zIndex = 10;
    }

    competence[i].onmouseout = function() {
    this.style.zIndex = this.dataset.index;
    }

    competence[i].dataset.index = i;
}

i 변수가 반복문의 블록 스코프에 한정되게하여 클로저 문제가 발생하지 않게 하였습다. 또한, dataset 속성을 사용하여 요소의 인덱스 값을 저장하여 onmouseout 이벤트 핸들러에서는 해당 요소의 dataset.index 값을 가져와서 zIndex 값을 복원하였습니다. 이로써 마우스가 요소 위로 올라갈 때 zIndex 값이 10으로 변경되며, 마우스가 요소를 벗어날 때는 해당 요소의 인덱스 값에 따라 원래 zIndex 값으로 복원되게 구현을 완료 하였습니다.

역량

C언어

기본적인 프로그래밍 지식과 기초를 다지는 데 큰 역할을 했습니다. C 언어를 통해 메모리 관리, 포인터 연산, 입출력 등의 핵심 개념과 프로그래밍 기법을 익혔습니다. 이를 통해 프로그래밍과 효율적인 알고리즘 설계에 대한 이해를 갖추었습니다.

JAVA

JAVA 언어를 통해 다양한 프로젝트 경험을 쌓아 객체지향 개념과 다양한 기능을 숙달했습니다. 다양한 도구와 라이브러리를 효과적으로 활용하여 개발 생산성을 높일 수 있습니다. 안드로이드 프로젝트 경험이 있으며 JAVA 기반 프레임워크인 Android Studio를 사용하여 안정적이고 효율적인 애플리케이션을 개발할 수 있습니다. UI/UX 디자인과 사용자 피드백을 고려하여 사용자가 편리하게 앱을 사용할 수 있는 경험을 제공하는 것을 중요시 여깁니다.

인문학

인문학적인 관점을 가지고 개발에 접근하는 것도 제 강점 중 하나입니다. 인문학적인 사고방식을 통해 사용자의 니즈와 감성을 이해하고, 디자인과 사용자 경험을 중요시하여 인간 중심적인 솔루션을 제공합니다. 이를 통해 기술적인 역량 뿐만 아니라 인간과의 상호작용을 고려한 효과적인 개발할 수 있습니다.

네트워크

프로토콜과 통신 방식을 이해하고, 소켓 프로그래밍을 통해 네트워크 통신을 구현할 수 있습니다. 이를 통해 안정적이고 효율적인 데이터 교환을 지원하는 애플리케이션을 개발할 수 있습니다.

자료구조

효율적으로 설계하고 알고리즘을 구현하여 데이터를 효과적으로 처리하는 데 능숙합니다.

데이터베이스

데이터베이스에 대한 지식 역시 제 강점 중 하나입니다. 데이터베이스 설계, 쿼리 작성, 성능 최적화 등을 다룰 수 있으며, SQL을 활용하여 데이터를 효율적으로 관리하고 분석할 수 있습니다. 데이터베이스의 중요성을 이해하고, 데이터의 보안과 무결성을 유지하며 효율적인 데이터 처리할 수 있습니다.

Xcode 14

Xcode 14에는 모든 Apple 플랫폼용 앱을 개발, 테스트, 배포하는 데 필요한 모든 것이 갖추어져 있습니다. 새로운 멀티 플랫폼 앱 경험을 통해 Swift 및 SwiftUI의 간결함과 뛰어난 성능을 활용하고, 향상된 에디터 기능으로 더 빨리 코딩할 수 있으며, Xcode Cloud에서 테스팅을 시작하여 TestFlight 및 App Store로 배포가 가능합니다. 이제 멋진 앱을 더욱 간편하게 개발할 수 있습니다.

더 가볍게. 더 빠르게.

Xcode 14의 바이너리가 이전 버전보다 30% 작아져 더 빠르게 시작할 수 있습니다. 이제 Xcode에서 watchOS 및 tvOS용 시뮬레이터 런타임을 다운로드할 수 있어 최신 플랫폼을 원하는 대로 사용할 수 있습니다. 모든 빌드 및 링크 단계에서 병렬성이 개선되어 프로젝트를 최대 25% 더 빠르게 빌드할 수 있습니다. 새로운 빌드 타임라인으로 예상외로 오래 걸리는 빌드 작업 및 빌드 내의 병목 현상을 식별할 수 있습니다.

멀티 플랫폼을 위한 설계

새로운 멀티 플랫폼 타깃에서 iOS, iPadOS, macOS 및 tvOS 전반에서 사용 가능한 단일 SwiftUI 인터페이스를 생성합니다. 코드의 유지보수성이 개선되었으며 각 플랫폼의 고유 기능을 활용하기 위해 코드를 맞춤화할 수도 있습니다. 애셋 카탈로그의 새로운 앱 아이콘은 개발자가 모든 플랫폼용 단일 아이콘 크기를 제공하면 Xcode에서 나머지 모두를 생성하는 경험을 제공합니다.

생산적인 코딩

개선된 많은 언어 및 편집 기능을 사용하여 생산성을 더욱 높일 수 있습니다. 더 스마트한 코드 완성 및 추가적인 동적 스니펫으로 더 빠르게 원하는 코드를 작성할 수 있습니다. 스크롤하는 동안 코드 구조(예: 함수 선언)가 표시되어 어디에서 작업하는지 바로 알 수 있습니다. 또한 Swift의 정규 표현식에 구문 강조, 리팩터링 작업 등이 통합되었습니다.

Swift, SwiftUI, Swift 패키지

Swift, SwiftUI 및 Xcode 14가 일심동체가 되어 작동합니다. SwiftUI 미리보기는 대화식으로 바로 사용할 수 있으며 클릭 한 번으로 라이트 모드 및 다크 모드와 같은 UI 변형을 만들 수 있습니다. Instruments용 새로운 템플릿을 통해서는 배포된 Actor 및 다른 Swift 동시 실행 기능의 사용 정보를 손쉽게 디버그 및 최적화할 수 있습니다. 또한 빌드 및 명령어 패키지 플러그인을 통해 Xcode 및 빌드 프로세스를 새로운 방식으로 맞춤화할 수 있습니다.

Swift

손쉽게 학습할 수 있는 강력한 프로그래밍 언어.

Swift는 iOS, iPadOS, macOS, tvOS 및 watchOS를 위한 강력하고 직관적인 프로그래밍 언어입니다. Swift 코드 작성은 대화식으로 재미있고, 구문은 간결하면서도 표현력이 풍부하며, Swift에는 개발자들이 좋아하는 첨단 기능이 포함되어 있습니다. Swift 코드는 안전하게 설계되었으며 빛의 속도로 빠르게 실행되는 소프트웨어를 제작할 수 있습니다.

최신 기능

Swift는 프로그래밍 언어에 대한 최신 연구 결과이며 수십 년에 걸친 Apple 플랫폼 구축 경험이 결합되어 있습니다. 명명된 매개변수는 Swift로 작성된 API를 더 쉽게 읽고 유지 관리할 수 있게 해주는 명확한 구문으로 표현됩니다. 세미콜론도 입력할 필요가 없습니다. 추론된 타입은 코드를 더 명확하게 만들고 실수를 덜 하도록 도와주며, 모듈은 헤더를 없애고 이름 공간을 제공합니다. 문자열은 다국어 및 이모티콘 지원을 위해 유니코드 표준을 따르며 다양한 사례에 사용할 수 있도록 성능을 최적화하기 위해 UTF-8 기반의 인코딩을 사용합니다. 엄격하고 확정적인 참조 계수를 사용하고, 가비지 컬렉션 관련 오버헤드 없이 최소한의 메모리 사용량을 유지하여 메모리를 자동으로 관리합니다. 비동기식 동작을 정의하는 간단한 내장 키워드를 사용하여 동시성 코드를 작성할 수 있어 코드의 가독성이 높아지고 오류 가능성이 낮아집니다.

안전 중심 설계

Swift는 불안전한 코드의 전체 클래스를 제거합니다. 변수는 사용 전에 항상 초기화되고, 배열 및 정수에 대한 오버플로우 검사가 수행되며, 메모리는 자동으로 관리됩니다. 또한 메모리에 대한 독점적인 접근을 통해 많은 프로그래밍 실수가 발생하지 않도록 보호합니다. 개발자의 의도를 쉽게 정의할 수 있도록 구문이 조정됩니다. 예를 들어 세자로 된 간단한 키워드를 사용하여 변수(var) 또는 상수(let)를 정의합니다. 뿐만 아니라 Swift는 특히 배열 및 딕셔너리와 같이 일반적으로 사용되는 값 타입을 최대한 활용합니다. 이는 해당 타입으로 사본을 만들 경우 다른 곳에서 수정할 수 없다는 것을 의미합니다. 또 다른 안전 기능은 기본적으로 Swift 객체가 결코 nil이 될 수 없게 하는 것입니다. 실제로 Swift 컴파일러는 컴파일 시 오류가 있는 nil 객체를 만들거나 사용할 수 없도록 합니다. 이렇게 하면 코드를 훨씬 깔끔하고 안전하게 작성할 수 있으며 앱에서 거대한 카테고리의 런타임 충돌을 방지할 수 있습니다. 하지만 nil이 유효하고 적절한 경우도 있습니다. 이러한 경우를 위해 Swift는 선택 사항이라는 혁신적인 기능을 제공합니다. 선택 사항에는 nil이 포함될 수 있지만 Swift 구문은 ? 구문을 사용하여 nil을 안전하게 처리하도록 함으로써 컴파일러에 동작을 이해하고 안전하게 처리될 것임을 표시합니다.

탁월한 제1언어

Swift는 코딩의 세계로 들어가기 위한 관문입니다. 실제로 Swift는 아직 공부 중인 학생이나 새로운 진로를 탐색하는 사람, 누구나 사용할 수 있는 최초의 프로그래밍 언어로 개발되었습니다. Apple은 교육자들을 위해 강의실 안팎에서 Swift를 가르치기 위한 무료 커리큘럼을 개설했습니다. 코딩 입문자들은 Swift 코드를 사용하여 대화식으로 즐겁게 시작할 수 있는 iPad용 앱인 Swift Playgrounds를 다운로드할 수 있습니다.
앱 개발자 지망생들은 무료 과정을 통해 Xcode로 첫 번째 앱을 빌드하는 방법을 학습할 수 있습니다. 뿐만 아니라 전 세계의 Apple Store에서는 Swift 코드를 실질적으로 경험해 볼 수 있는 Today at Apple 코딩 및 앱 세션을 주최합니다.

오픈 소스

Swift는 Swift.org에서 오픈 소스로 개발되었으며 모든 사람이 이용할 수 있는 소스 코드, 버그 추적기, 포럼 및 정기적인 개발 빌드가 포함되어 있습니다. Apple뿐만 아니라 수백 명의 외부 개발자가 참여하는 이 광범위한 개발자 커뮤니티에서 Swift를 더욱 멋진 언어로 만들기 위해 협력하고 있습니다. 이 커뮤니티에 속한 개발자들이 Swift의 놀라운 잠재력을 실현한 경험을 공유하는 더 광범위한 블로그, 팟캐스트, 컨퍼런스 및 모임도 있습니다.

크로스 플랫폼

Swift는 Linux를 비롯한 모든 Apple 플랫폼을 지원하며, 더 많은 플랫폼을 지원하기 위해 커뮤니티 회원들이 활발하게 활동하고 있습니다. 커뮤니티에서는 SourceKit-LSP를 사용하여 Swift 지원을 다양한 개발자 도구로 통합하는 작업도 수행합니다. Apple은 Swift로 소프트웨어를 더 빠르고 안전하게 만드는 동시에 프로그래밍을 더 재미있게 만드는 다양한 방법을 모색할 수 있어 기쁘게 생각합니다.

서버를 위한 Swift

Swift는 Apple 플랫폼에서 수많은 신규 앱을 강력하게 지원하는 동시에 새로운 클래스의 최신 서버 응용 프로그램에도 사용됩니다. Swift는 런타임 안정성, 컴파일된 성능 및 소규모 메모리 공간이 필요한 서버 앱에 사용하기에 적합합니다. 커뮤니티에서는 Swift를 서버 응용 프로그램 개발 및 배포용으로 활용할 수 있도록 Swift 서버 작업 그룹을 구성했습니다. 이러한 노력의 첫 번째 성과로 고성능 프로토콜 서버 및 클라이언트를 위한 크로스 플랫폼 비동기식 이벤트 중심 네트워크 응용 프로그램 프레임워크인 SwiftNIO가 탄생했습니다. SwiftNIO는 로깅, 메트릭 및 데이터베이스 드라이버를 비롯한 추가 서버 지향 도구 및 기술 구축을 위한 기반으로, 현재 모두 활발하게 개발이 진행 중입니다.

Playgrounds 및 REPL (Read-Eval-Print-Loop)

iPad용 Swift Playgrounds와 마찬가지로 Xcode의 Playground를 사용하여 매우 간단하고 재미있게 Swift 코드를 작성할 수 있으며, 한 줄의 코드를 입력하면 결과가 즉시 나타납니다. 작성 후에는 코드 옆에서 Quick Look 기능으로 결과를 확인하거나 해당 결과를 바로 아래에 고정할 수 있습니다. 결과 보기에 그래픽, 결과 목록 또는 시간 경과에 따른 값의 그래프를 표시할 수 있습니다. Timeline Assistant를 열면 복잡한 뷰가 변하고 움직이는 것을 볼 수 있어 새로운 UI 코드를 실험하는 데 유용하며, 코드를 작성하면서 움직이는 SpriteKit 장면을 재생할 수 있습니다. 플레이그라운드에서 코드를 완벽하게 만들었다면 코드를 프로젝트로 이동하기만 하면 됩니다. Swift는 터미널 또는 Xcode의 LLDB 디버깅 콘솔에서도 상호 작용이 가능합니다. Swift 구문을 사용하여 실행 중인 앱을 평가하고 이러한 앱과 상호 작용하거나, 새로운 코드를 작성하여 스크립트와 유사한 환경에서 어떻게 작동하는지 확인할 수 있습니다.

Package Manager

Swift Package Manager는 Swift 라이브러리 및 실행 파일을 구축, 실행, 테스트 및 패키징하기 위한 단일 크로스 플랫폼 도구입니다. Swift 패키지는 Swift 커뮤니티에 라이브러리 및 소스 코드를 배포하는 가장 좋은 방법입니다. 패키지 구성은 Swift로 직접 작성되어 손쉽게 대상을 구성하고, 제품을 결정하며, 패키지 종속성을 관리할 수 있습니다. 또한 Swift 패키지에는 프로젝트 빌드 및 추가 도구 제공에 도움이 되는 맞춤형 명령어를 포함할 수 있습니다. Swift Package Manager는 실제로 Swift로 빌드되며 Swift 오픈 소스 프로젝트에 패키지 형태로 포함됩니다.

Objective-C 상호 운용성

지금 바로 Swift를 사용하여 전혀 새로운 응용 프로그램을 만들거나 Swift 코드 사용을 시작하여 앱에 새로운 기능을 구현할 수 있습니다. 동일한 프로젝트에서 기존 Objective-C 파일과 함께 Swift 코드가 공존하며, Objective-C API에 대한 완전한 접근 권한이 제공되므로 적용이 용이합니다.
